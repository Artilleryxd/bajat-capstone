# FinSight AI — DB Guidelines

## Migration

Run `backend/db/migrations/002_loans_tables.sql` in the Supabase SQL editor before using the loans feature.

## Tables

### `loans`
Stores each user's active loans.

| Column | Description |
|--------|-------------|
| principal | Original loan amount |
| rate | Interest rate |
| EMI | Equated Monthly Instalment |
| tenure | Loan duration |

### `loan_strategies`
Stores optimization results.

| Column | Type | Description |
|--------|------|-------------|
| `strategy_data.full_schedule` | JSONB | Complete month-by-month amortization schedule |
| AI rationale | text | Plain-language explanation generated by AI |

## Row-Level Security (RLS)

Both tables have RLS enabled with a `user_isolation` policy — users can only access their own rows.

## Storage Notes

- The full simulation schedule (up to 600 months) is stored in `loan_strategies.strategy_data.full_schedule` as JSONB.
- Only the **first 24 months** of the schedule are returned to the frontend via API.
- Yearly timeline data is aggregated separately for chart rendering.
